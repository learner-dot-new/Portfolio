01 Engine Overview

The core infrastructure handles bipartite matching between users and captains. It implements a **Location-Aware Dispatch Logic** that calculates real-time proximity and vehicle-specific fare matrices.

Geospatial Search

Locates captains using the $geoNear MongoDB aggregation for minimal latency in high-density areas.

Signal Clustering

Broadcasts 'new-ride' events exclusively to offline captains within a specific geographical shard.
